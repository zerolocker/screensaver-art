- **2026-07-18** — Fixed slow/flaky "Relaunch to update" banner (diagnosed from the installed app's own main.log): (1) checks now also fire on window focus (≥15 min apart) and on system wake — the 6h `setInterval` was the only trigger and doesn't tick during sleep (observed 6h42m gaps); (2) `/updates` resolves the latest GitHub release with `cache: 'no-store'` — the 120s stale-while-revalidate cache served a 17-minute-stale "latest" right after a publish; (3) once an update is downloaded, re-checks are skipped and `ready` is sticky in the reducer — each 6h re-check used to hide the banner and re-download the full ~190MB zip (~30 redundant downloads of 1.4.5 over 8 days). Updater logs now carry the check trigger + timings so time-to-banner is measurable per release.
